Background

In the modern world, robots are often necessary in certain environments such as hospitals, factories, tours, etc. Robots can replace humans with various and even complex tasks, some of which will be difficult. Therefore, from a robotic perspective, it is necessary to think and plan trajectories, avoiding obstacles in space and the like in order for the robot to successfully perform its mission.

Swarm Lifting is a system that provides solutions for finding paths in an environment that includes obstacles, where robots need to navigate and identify objects in the environment and move them into one place while interacting with each other.

System description

Our system will simulate factory robots, the simulation will include obstacles, different routes and several robots, so that each will be responsible for a task that will be navigating towards an object (eg crate) and moving it from one place to another. Each robot goes from source to target, it needs to navigate correctly in a factory that will include obstacles and carry out its mission. For the navigation task we will look at different algorithms including genetic algorithms.

Our system will use an operating system called ROS, which provides tools and libraries for obtaining, building and writing code for the robot. Through ROS the robots can communicate with the environment in which they work, and in general in the Gazebo simulator that accurately and effectively simulated the population of robots. The robots will use sensors such as distance sensor (2 / D 3D), depth sensor, bumper sensor (obstacle avoidance) and their own camera

ROS is a unique operating system for robots. It provides a number of services such as hardware abstraction, low-level device control, widespread functionality, messaging between processes, and package management. It also provides tools and libraries for obtaining, building, writing and executing code. This system is a framework designed to simplify the task of creating complex robot behavior across a wide range of robotic platforms - it can be through simulation or a real robot.

Through, ROS we can see and illustrate the communication of the robots to the environment we talked about, such as data transfer, etc. Communication between system components is peer-to-peer communication.

The simulator we use is called Gazebo. Gazebo is a three-dimensional dynamic simulator with the ability to accurately and effectively simulate populations of robots in indoor and outdoor environments. Like gaming engines, Gazebo offers a higher-fidelity physics simulation, a suite of sensors, user interfaces, and programs.

Typical uses of Gazebo: • Testing robotic algorithms. • Robot design. • Performing regression testing using realistic scenarios.

Key features of Gazebo: • Rich library of models and robot environments. • Wide range of sensors. • Convenient programmatic and graphical interfaces.

In our system we will use a robot called turtlebot. The simulator we use can provide this model of the robot. The "stack" navigation will take as input the current location of the robot, its desired location (destination) and data the robot has such as sensors and the like.

Our robot can run localization and mapping algorithms, and an arm can be attached to it to manipulate objects. Among other things, it includes a distance sensor and a camera.

Global path planning problem of a mobile robot is essential to give the robot the ability to plan a collision-free path between two locations. Indeed, in any mobile robot applications, the robot must be able to ﬁnd its path to a certain goal position while avoiding obstacles and reducing the path cost

The path planning problem is inﬂuenced by two factors:

1. The environment, which can be static or dynamic. A static environment is unchanging, the start and goal positions are ﬁxed, and obstacles do not change locations over time. However, in a dynamic environment, mobile robots are exposed to unexpected situations, including the locations of obstacles and the target, which may change over time. 2. The knowledge that the robot has about the environment; if the robot has a complete knowledge about the environment, this problem is known as global path planning. On the other hand, if the robot has an incomplete knowledge, this problem is classiﬁed as local path planning.

ROS Manipulation is the term used to refer to any robot that manipulates something in its environment. And what does this mean? Well, it means that it physically alters something in the world, for instance, changing it from its initial position.
